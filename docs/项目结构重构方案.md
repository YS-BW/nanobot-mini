# 项目结构重构方案

> 只保留当前有效的结构方案。旧分析、旧路线图、旧解释文档均已废弃。

## 当前结构

当前代码是平铺结构：

```text
bananabot/
├── __main__.py
├── config.py
├── context.py
├── llm.py
├── memory.py
├── runner.py
├── session/
├── tools/
├── tools.py
└── types.py
```

当前结构的问题：

- 入口层、运行时、状态层耦合
- prompt 和 context 混在一起
- `llm.py` 职责过宽
- `session / compact / memory` 没有收敛到统一子域
- `tools.py` 与 `tools/` 冲突
- 没有统一的应用服务接口
- 没有统一的请求/响应/事件结构

## 各层职责

- `app`
  - 入口、交互、依赖组装、统一服务接口、对外契约

- `runtime`
  - 运行时主循环、上下文装配、prompt、事件生成

- `memory`
  - `session/history/summary/MEMORY` 的状态与 compact 策略

- `llm`
  - 模型调用协议、返回结构、错误语义

- `tools`
  - 工具抽象、工具注册、工具实现

- `infra`
  - 配置、日志、路径、环境

## 统一接口

第一阶段已预留：

```python
async def chat(request: ChatRequest) -> ChatResponse
async def chat_stream(request: ChatRequest) -> AsyncIterator[AgentEvent]
```

统一契约定义在 `app/contracts.py`：

```python
@dataclass
class ChatRequest:
    session_id: str
    user_input: str
    metadata: dict | None = None


@dataclass
class ChatResponse:
    session_id: str
    message: str
    finish_reason: str = "stop"


@dataclass
class AgentEvent:
    type: str
    session_id: str
    message: str | None = None
    data: dict | None = None
```
