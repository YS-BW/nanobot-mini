# 仓库总览

## 定位

`bananabot` 当前是一个终端优先的 Agent 内核。
现在仓库的主线已经统一成 `thread / task_run / turn / step`，并围绕这条主线组织应用层、运行时、记忆层、模型层和工具层。

这意味着后续无论接 TUI、桌面端还是 Web 端，核心执行链都应该继续复用同一套服务接口和事件流，而不是再为不同端各写一套对话内核。

## 顶层结构

```text
bananabot/
├── bananabot/
│   ├── app/
│   ├── runtime/
│   ├── memory/

├── TODO.md
└── models.toml
```

## 模块分工

- `app/`：统一请求入口、服务编排、TUI
- `runtime/`：运行时模型、事件、状态推进、主循环、上下文
- `memory/`：thread 消息目录、compact、长期记忆、working memory
- `llm/`：模型档案、provider 规则、HTTP 客户端
- `tools/`：工具抽象、注册表、内置工具
- `infra/`：配置、日志、路径、runtime 状态存储

## 主执行链路

1. `app/bootstrap.py` 组装依赖
2. `app/service.py` 接收 `TaskRequest`
3. 用户消息写入 thread 当前窗口和历史日志
4. `runtime/context_engine.py` 组装模型消息
5. `runtime/runner.py` 运行模型与工具循环
6. `runtime/coordinator.py` 推进 `thread / task_run / turn / step`
7. `infra/runtime_state_store.py` 持续写状态和事件
8. `memory/working_memory.py` 回写结构化短期记忆
9. `memory/compact_service.py` 压缩旧消息并整合记忆
10. `app/cli.py` 消费统一事件流并渲染界面

## 当前存储

### thread 消息目录

默认路径：`~/.bananabot/sessions/<thread_id>/`

- `session.jsonl`：当前窗口消息
- `history.jsonl`：完整历史
- `summary.jsonl`：compact 摘要
- `MEMORY.md`：thread 长期记忆

### runtime 状态目录

默认路径：`~/.bananabot/runtime-state/`

- `threads/<thread_id>.json`
- `task_runs/<task_run_id>.json`
- `turns/<turn_id>.json`
- `steps/<step_id>.json`
- `event_log.jsonl`

## 当前边界

已稳定下来的部分：

- 应用层只走 `TaskRequest / TaskResponse / AgentEvent`
- runtime 事件只走 `EventEnvelope.payload`
- TUI 已经接到统一事件流
- working memory 和 compact 已经进入主链
- runtime 状态落盘已经可测试、可观察

还会继续扩展但暂时没彻底做深的部分：

- provider 能力矩阵与模型策略
- skill 机制
- 审批、恢复、沙箱和安全策略
- 桌面端 / Web 端接入
