> BananaBot 的分层记忆与上下文管理方案

## 一、核心问题

对话越来越长，token 不够用怎么办？

---

## 二、存储结构

每个 session 有一套独立的存储文件，通过 **session_id** 隔离：

```
~/.bananabot/sessions/<session_id>/
├── session.jsonl    # 当前会话上下文（可被 compact 裁切）
├── history.jsonl     # 已被 compact 的对话历史（append-only，不用于上下文）
├── summary.jsonl    # 第一轮 compact 产物（累积，用于第二轮）
└── MEMORY.md        # 第二轮 compact 产物（长期记忆）

~/.bananabot/projects/<cwd-hash>/memory/  # 项目级记忆（独立于 session）
```

### 各文件职责

| 文件 | 职责 | 写入方式 | 用于上下文 |
|------|------|---------|-----------|
| **session.jsonl** | 当前工作上下文 | 覆盖（compact 后保留最近消息） | ✅ |
| **history.jsonl** | 完整对话存档 | append-only | ❌ |
| **summary.jsonl** | 第一轮 compact 摘要 | append | ✅（注入系统提示词） |
| **MEMORY.md** | 第二轮 compact 结构化记忆 | 覆盖写入 | ✅（注入系统提示词） |

## 三、Compact 机制

### 3.1 阈值设计

按 **上下文窗口比例** 设定，非固定数值：

| 阶段 | 阈值 | 说明 |
|------|------|------|
| 第一轮 compact | `context_window * 0.70` | session 超过 70% 触发 |
| 第二轮 compact | `context_window * 0.85` | summary 超过 85% 触发 |
| safety buffer | `context_window * 0.15` | 留给 LLM 输出空间 |

### 3.2 第一轮 Compact（session → history + summary）

**触发条件**：session token 超过 `context_window * 0.70`

**操作流程**：
1. 检测到 token 超阈值
2. 保留最近 20 条消息，其余裁剪
3. 裁剪下来的完整消息 → 追加写入 history.jsonl
4. 调用 LLM 对裁剪内容进行总结（要点：项目设置、模块状态、重要决策、已知问题）
5. 总结结果追加写入 summary.jsonl
6. 失败重试 5 次，仍失败则消息写入 history.jsonl（不丢数据）

**summary.jsonl 格式**：
```jsonl
{"id": "uuid", "timestamp": "2026-04-15 10:30", "summary": "...", "version": 1}
```

### 3.3 第二轮 Compact（summary → MEMORY.md）

**触发条件**：summary token 超过 `context_window * 0.85`

**操作流程**：
1. 检测到 summary 超过阈值
2. 读取 summary.jsonl（所有条目）和现有 MEMORY.md
3. 调用 LLM 整合成新的 MEMORY.md（合并重复、更新过时、保持简洁）
4. 新的 MEMORY.md 覆盖写入
5. 清空 summary.jsonl

## 四、上下文注入顺序

每次对话开始时，上下文按以下顺序拼接：

```
┌─────────────────────────────────────────────────────────┐
│ ROLE: system                                            │
├─────────────────────────────────────────────────────────┤
│ ## 全局指令                                             │
│ （~/.bananabot/BANANA.md 内容）                        │
│                                                         │
│ ## 项目指令                                             │
│ （从 workspace 向上查找的 BANANA.md）                    │
│                                                         │
│ ## 记忆                                                │
│ （MEMORY.md - 第二轮 compact 产物）                     │
│ ---                                                    │
│                                                         │
│ ## 近期会话摘要                                         │
│ （summary.jsonl - 第一轮 compact 产物）                  │
│ ---                                                    │
│                                                         │
│ ## 当前会话                                            │
│ （session.jsonl - 当前工作上下文，取最近 20 条）         │
│ ---                                                    │
│                                                         │
│ # 🍌 BananaBot                                         │
│ [系统提示词模板]                                        │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│ ROLE: user                                             │
├─────────────────────────────────────────────────────────┤
│ [用户输入]                                             │
└─────────────────────────────────────────────────────────┘
```

**关键点**：
- history.jsonl 不用于上下文，仅作备份
- session.jsonl 只保留最近消息（被 compact 裁切的部分进入 history + summary）
- summary 和 MEMORY 都在系统提示词里

## 五、Compact 触发方式

| 方式 | 触发条件 | 执行位置 |
|------|---------|---------|
| 自动 compact | session token > 70% | chat_once 结束时检查 |
| 手动 compact | `/compact` 命令 | 交互模式命令 |

## 六、Session 管理

### 6.1 Session 类职责

```python
class Session:
    def add(role, content, **kwargs)          # 添加消息
    def save()                               # 保存到 session.jsonl
    def reload()                             # 从 session.jsonl 重新加载
    def compact(context_window, threshold)     # 执行 compact
    def estimate_tokens()                    # 估算 token
    def append_history(messages)              # 追加到 history.jsonl
    def append_summary(summary)               # 追加到 summary.jsonl
```

### 6.2 SessionManager 职责

```python
class SessionManager:
    def get_or_create(key)                   # 获取或创建会话
    def list_sessions()                       # 列出所有会话
    def delete(key)                           # 删除会话
```

## 七、Memory 管理

### 7.1 MemoryStore 类职责

```python
class MemoryStore:
    def get_memory_context() -> str | None   # 获取记忆（前 200 行）
    def save_memory(content)                  # 保存记忆
    @staticmethod
    def find_banana_md(workspace)             # 查找 BANANA.md
```

### 7.2 项目隔离

按 workspace hash 隔离：
```python
project_hash = hashlib.md5(str(workspace.resolve()).encode()).hexdigest()[:8]
memory_dir = global_dir / "projects" / project_hash / "memory"
```

## 八、错误处理

| 错误 | 处理方式 |
|------|---------|
| LLM compact 失败 | 重试 5 次 |
| 5 次都失败 | 报错提示，原生消息写入 history.jsonl（不丢数据）|
| 单条 summary 超过 500 tokens | 拆成多条 |
